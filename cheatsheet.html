Ch 5Eigenvalues + Eigenvectors
Lay §5.1 · ⭐⭐⭐ HIGH
⭐

Definition: v ≠ 0 is an eigenvector of A with eigenvalue λ if

A v = λ v

Eigenspace for λ = Nul(A − λI) — solve (A − λI)x = 0

  • Triangular matrix: eigenvalues = diagonal entries (free win)
  • 0 is an eigenvalue ⇔ A is singular ⇔ det A = 0
  • Eigenvectors for distinct eigenvalues are linearly independent
  • Eigenspace dimension ≥ 1 (since at least one eigenvector exists)
VERIFY v is eigenvector:
  1. Compute Av
  2. Check if Av = λv for some scalar λ
Ch 5Characteristic Equation
Lay §5.2 · ⭐⭐⭐ HIGH
⭐
det(A − λI) = 0

Solve for λ → those are the eigenvalues.

  • Sum of eigenvalues = trace(A) (sum of diagonal) — sanity check
  • Product of eigenvalues = det(A) — sanity check
  • Algebraic multiplicity: how many times λ appears as root of char poly
  • Geometric multiplicity: dim(eigenspace) = # free vars in (A−λI)x=0
  • Always: 1 ≤ geo. mult. ≤ alg. mult.
  • Similar matrices have same char poly, eigenvalues, trace, det
For 2×2: char poly = λ² − (trace)λ + det.
For 3×3: cofactor expand det(A−λI) along row/col with most zeros.
Ch 5Diagonalization
Lay §5.3 · ⭐⭐⭐ HIGH
⭐
A = P D P⁻¹

P = columns are eigenvectors. D = diagonal of eigenvalues (same order!)

4-STEP DIAGONALIZE:
  1. Find eigenvalues from det(A−λI) = 0
  2. For each λ, find basis of Nul(A−λI) → eigenvectors
  3. If you have n linearly independent eigenvectors → build P = [v₁ | v₂ | … | vₙ]
  4. Build D = diag(λ₁, λ₂, …, λₙ) — same order as P columns

Power formula (exam-favorite):

Ak = P Dk P⁻¹

Dk just raises diagonal entries to power k.

  • Diagonalizable iff n linearly independent eigenvectors
  • n distinct eigenvalues ⇒ automatically diagonalizable
  • Diagonalizable iff geo.mult. = alg.mult. for every λ
Ch 5Markov Chains (class addition)
Chap5_Lectures_notes · ⭐⭐⭐ HIGH
⭐

Stochastic matrix M: columns sum to 1, all entries ≥ 0.

State transition: xk+1 = M·xk

Steady state p (long-run probability):

M p = p ⇔ (M − I) p = 0
FIND STEADY STATE:
  1. Verify M is stochastic (col sums = 1)
  2. Compute M − I
  3. Solve (M−I)p = 0 — row-reduce → one free variable
  4. Apply normalization: Σpᵢ = 1 → solves the free var
  5. p is the steady-state probability vector

Example 2-state: M = [[0.7, 0.3], [0.4, 0.6]]
M − I = [[−0.3, 0.3], [0.4, −0.4]] → 4p₁ − 4p₂ = 0 (one eq) → p₁ = p₂.
With p₁ + p₂ = 1: p = (0.5, 0.5)ᵀ.

If M is regular (Mᵏ all positive for some k) → unique steady state, xk → p regardless of x₀.
LPFormulation + Graphical
Budnick L17–18 · ⭐⭐⭐ HIGH
⭐

LP elements:

  • Decision vars (what you choose)
  • Objective function: max profit or min cost
  • Constraints (≤, ≥, =)
  • Non-negativity: xᵢ ≥ 0
GRAPHICAL METHOD (2 vars only):
  1. Treat each constraint as line (replace ≤ with =, plot)
  2. Shade feasible side of each line
  3. Feasible region = intersection of all shaded
  4. Find ALL corner points (intersections of constraint lines)
  5. Evaluate Z at each corner
  6. Pick max (or min)
  • Unbounded: feasible region extends to infinity in obj direction → no optimal
  • Infeasible: no point satisfies all constraints → no solution
  • Multiple optima: obj parallel to a binding edge → entire edge optimal
LPSimplex Method
Budnick L20–21 · ⭐⭐⭐ HIGH
⭐

Standard form (max): max Z = cᵀx, subject to Ax ≤ b, x ≥ 0.

  • ≤ constraint: add slack variable s (≥0)
  • ≥ constraint: subtract surplus + add artificial
  • = constraint: add artificial
SIMPLEX ALGORITHM (max):
  1. Entering var: column with most negative Z-row coefficient. If all ≥ 0 → STOP, optimal.
  2. Leaving var: smallest non-negative ratio bᵢ / aᵢⱼ (only where aᵢⱼ > 0). If no positive aᵢⱼ → UNBOUNDED.
  3. Pivot: make pivot entry = 1 by dividing pivot row. Then zero out the rest of the entering column via row ops.
  4. Repeat from step 1.

Read solution: basic vars = RHS values; non-basics = 0.

For min: choose most POSITIVE Z-row coeff to enter (or convert to max by negating Z).
LPDual Problem
Budnick L23–24 · MED
🟡

For primal: max cᵀx, Ax ≤ b, x ≥ 0 — its dual is:

min bᵀy, Aᵀy ≥ c, y ≥ 0
  • m constraints, n vars in primal ↔ m vars, n constraints in dual
  • Aᵀ swaps rows ↔ cols
  • Constraint sense flips: primal ≤ ↔ dual ≥
  • Weak duality: cᵀx ≤ bᵀy (any feasible pair)
  • Strong duality: at optimum, cᵀx* = bᵀy*

Dual variables y* = shadow prices (marginal value of each resource).

TRTransportation — Initial BFS
Budnick L25–26 · ⭐⭐ MED
🟡

Balance first: if total supply ≠ total demand → add dummy row (extra supply) or dummy column (extra demand) with cost = 0.

Must allocate m + n − 1 cells. If fewer → degenerate, use ε trick.

NWCM (Northwest Corner):
  1. Start at top-left cell
  2. Allocate min(supply, demand) there
  3. Eliminate row or col that's fully used
  4. Move down or right, repeat
⭐ VAM (Vogel's Approx — exam favorite):
  1. For each row and col: compute penalty = (2nd lowest cost) − (lowest cost)
  2. Choose row/col with highest penalty
  3. Allocate to lowest-cost cell in that row/col: min(supply, demand)
  4. Eliminate satisfied row/col
  5. Recompute penalties for what remains, repeat
Least Cost: just always pick the cheapest available cell. Faster than NWCM but worse than VAM.
TRMODI Optimality
Budnick L26+ · MED
🟡
MODI ALGORITHM:
  1. Set u₁ = 0
  2. For each filled cell (i,j): uᵢ + vⱼ = cᵢⱼ → solve all uᵢ, vⱼ
  3. For each empty cell (i,j): compute rᵢⱼ = cᵢⱼ − (uᵢ + vⱼ)
  4. If all rᵢⱼ ≥ 0 → OPTIMAL
  5. Otherwise: pick most negative rᵢⱼ as entering cell
CLOSED LOOP REALLOCATION:
  1. Start at entering cell with +
  2. Draw rectangular loop through filled cells, alternating + − + −
  3. θ = min of allocations at − cells
  4. Add θ to + cells, subtract θ from − cells
  5. Re-do MODI check on new allocation
TRHungarian / Assignment
Budnick L27–28 · ⭐⭐ MED
🟡

Pre-process:

  • Must be n × n. Add dummy row/col with 0 if unbalanced.
  • For maximization: subtract every entry from max value. Now min the new matrix.
HUNGARIAN METHOD:
  1. Row reduce: subtract each row's min from that row
  2. Col reduce: subtract each col's min from that col
  3. Cover all zeros with minimum # of horizontal + vertical lines
  4. If lines = n → optimal. Pick zeros (one per row, one per col).
  5. If lines < n:
    • Find smallest uncovered entry e
    • Subtract e from all uncovered entries
    • Add e to all double-covered entries (where two lines cross)
    • Other entries unchanged
    • Go back to step 3
Ch 2Matrix Ops + Inverse + IMT
Lay §2.1–2.3 · MED
🟡

2×2 inverse:

A⁻¹ = (1 / (ad−bc)) · [[d, −b], [−c, a]]

3×3+ inverse: row-reduce [A | I] → [I | A⁻¹]. If can't reach I → A not invertible.

  • (AB)⁻¹ = B⁻¹A⁻¹ (reverse order!)
  • (Aᵀ)⁻¹ = (A⁻¹)ᵀ
  • (A⁻¹)⁻¹ = A
IMT — 14 equivalent statements (A is n×n):
The following are equivalent — if any one is true, all are:
  1. A is invertible
  2. A is row equivalent to Iₙ
  3. A has n pivot positions
  4. Ax = 0 has only the trivial solution
  5. Columns of A are linearly independent
  6. x → Ax is one-to-one
  7. Ax = b has a unique solution for every b ∈ Rⁿ
  8. Columns of A span Rⁿ
  9. x → Ax is onto Rⁿ
  10. ∃ matrix C such that CA = I
  11. ∃ matrix D such that AD = I
  12. Aᵀ is invertible
  13. Columns of A form a basis of Rⁿ
  14. det(A) ≠ 0
Ch 3Determinants
Lay §3.1–3.3 · MED
🟡

Cofactor expansion along any row or column:

det A = Σⱼ aij Cij, Cij = (−1)i+j det(Mij)

where Mij = (n−1)×(n−1) submatrix (delete row i, col j). Choose row/col with most zeros.

  • Triangular matrix: det = product of diagonal (FREE marks!)
  • det(AB) = det(A) · det(B)
  • det(Aᵀ) = det(A)
  • det(A⁻¹) = 1 / det(A)
  • If A has a row of zeros → det A = 0
  • If two rows are equal → det A = 0

Row operation effects on det:

  • Swap two rows: det × (−1)
  • Scale a row by k: det × k
  • Replace Rᵢ → Rᵢ + k·Rⱼ: det unchanged

Cramer's Rule (for invertible A, system Ax = b):

xi = det(Ai(b)) / det(A)

where Ai(b) = A with column i replaced by b.

Inverse via adjugate: A⁻¹ = (1/det A) · adj(A), adj(A) = transpose of cofactor matrix.

Ch 1Linear Systems (§1.1–1.5)
Lay · LOW–MED
🟡

REF (Row Echelon Form):

  • All zero rows at bottom
  • Leading entry of each row is to the right of leading entry of row above
  • Entries below each leading entry are 0

RREF (Reduced): also requires leading entry = 1, AND zeros above each leading 1.

Pivot positions: location of leading 1s in RREF. Pivot columns contain pivots; other columns correspond to free variables.

Consistency: a system is consistent iff RREF has no row [0 ... 0 | b] with b ≠ 0.

Parametric vector form (for ∞ solutions):

x = p + t1v1 + t2v2 + …

p = particular solution (set free vars = 0), vi = direction for each free variable.

Span{v₁, …, vₚ}: set of all linear combinations of these vectors.

Ax = b consistent ⇔ b ∈ Span(columns of A) ⇔ b is a linear combination of columns.

Ch 1LI + Transformations (§1.7–1.9)
Lay · MED
🟡

Linear Independence test:

Ax = 0 has ONLY the trivial solution

Row-reduce A. If no free variables (every column is pivot) → independent. Any free var → dependent.

  • If p > n vectors in Rⁿ → automatically dependent
  • Set containing 0 vector → always dependent

Linear Transformation T : Rⁿ → Rᵐ satisfies:

  • T(u + v) = T(u) + T(v)
  • T(c · u) = c · T(u)

Standard matrix of T:

A = [ T(e1) | T(e2) | … | T(en) ]

where eᵢ are standard basis vectors of Rⁿ.

T one-to-one ⇔ cols of A linearly independent ⇔ Ax = 0 only trivial

T onto ⇔ cols of A span Rᵐ ⇔ A has a pivot in every row

Common 2D transformations:

  • Rotation by θ: [[cosθ, −sinθ], [sinθ, cosθ]]
  • Reflection x-axis: [[1, 0], [0, −1]]
  • Reflection y-axis: [[−1, 0], [0, 1]]
  • Reflection y = x: [[0, 1], [1, 0]]
  • Projection onto x-axis: [[1, 0], [0, 0]]
  • Horizontal shear: [[1, k], [0, 1]]
🎯Exam Strategy · Test Day
Solve 6 of 7 × 4 marks · 3 hours · 30 min/question
🎯
FIRST 5 MINUTES (before writing anything):
  1. Read ALL 7 questions
  2. Mark each as STRONG / OK / WEAK
  3. Pick your 6 strongest. Skip the worst one.
  4. Roughly plan order: hardest while fresh OR easiest first for confidence (your call)
DURING (30 min per question):
  1. Show every step. A wrong final answer with right method usually gets 2–3 of 4 marks.
  2. Sanity check: sum of eigenvalues = trace; det = product of eigenvalues; M stochastic → col sums = 1; LP corner should be vertex of feasible region.
  3. If stuck > 5 min: move on. Come back later.
  4. Free wins to grab fast:
    • Triangular determinant = product of diagonal
    • Triangular matrix eigenvalues = diagonal entries
    • IMT — list 4 equivalences for 2 marks
    • Verify v is eigenvector by computing Av
    • Markov col sum check (instantly catches "is this stochastic?" Qs)
LAST 10 MINUTES:
  1. Recheck arithmetic + signs in computations
  2. Check you transcribed final answer correctly
  3. Make sure question numbers on your booklet match the Q you intended to answer
  4. If 5+ min left, attack the 7th question for partial credit
⚠ Common traps: sign errors in cofactor expansion (the (−1)i+j) · forgetting to normalize Markov steady state (Σpᵢ = 1) · LP simplex ratio test using rows where aij ≤ 0 · Hungarian with unbalanced matrix (need dummy row/col first) · MODI starting without setting u₁ = 0
